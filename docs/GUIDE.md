# دليل تشغيل نظام التفريغ الصوتي وصناعة التراجم (WhisperX & FastAPI & Next.js)

يوفر هذا الدليل شرحاً تفصيلياً متكاملاً لكيفية إعداد، وتثبيت وتثبيت خادم معالجة الذكاء الاصطناعي وإنتاج ملفات SRT, VTT, TXT محلياً وللإنتاج (Production).

---

## 🚀 متطلبات التشغيل الأساسية (Prerequisites)

1. **نظام التشغيل:** Linux (Ubuntu 20.04 أو أحدث) أو Windows 10/11 مع تفعيل WSL2.
2. **برامج أساسية:**
   - **Python:** الإصدار 3.10 أو 3.11 أو 3.12.
   - **Node.js:** الإصدار 18 أو أحدث مع مدير الحزم `npm` أو `yarn`.
   - **FFmpeg:** مثبت على خادم التشغيل ومعرف بالبيئة لتمكين استخراج الصوتيات.
   - **Docker & Docker Compose:** لإعداد الحاويات الفورية (موصى به للإنتاج).

---

## 🛠️ التشغيل المحلي المباشر (Local Setup without Docker)

إذا كنت ترغب بالتشغيل والتطوير المباشر على حاسوبك الشخصي:

### 1. إعداد وتطوير الـ Backend (FastAPI & WhisperX)

أولاً، تأكد من تثبيت عتاد FFmpeg في نظام التشغيلของคุณ:
- **في Ubuntu/Debian:** `sudo apt update && sudo apt install ffmpeg`
- **في macOS:** `brew install ffmpeg`
- **في Windows:** قم بتحميله وإضافته لـ System PATH.

تثبيت احتياجات Python وتدشين بيئة العمل:
```bash
# اذهب لمجلد الـ backend
cd backend

# أنشئ بيئة افتراضية
python -m venv venv
source venv/bin/activate  # في الويندوز: venv\Scripts\activate

# قم بتحديث pip وتثبيت المكتبات الأساسية
pip install -U pip setuptools wheel
pip install -r requirements.txt
```

لتشغيل السيرفر المحلي:
```bash
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```
من المفترض الآن أن تكون API نشطة على الرابط التالي: `http://localhost:8000` ويمكنك مطالعة توثيق Swagger التفاعلي عبر `http://localhost:8000/docs`.


### 2. إعداد وتطوير الـ Frontend (Next.js)

```bash
# اذهب لمجلد الـ frontend
cd frontend

# قم بتثبيت الحزم البرمجية
npm install

# قم بإنشاء ملف إعداد البيئة وتحديد مسار الـ API
echo "NEXT_PUBLIC_API_URL=http://localhost:8000" > .env.local

# ابدأ تشغيل سيرفر المطور المحلي
npm run dev
```
سيكون الموقع متاحاً للمعاينة وتجربة الترجمة على الرابط `http://localhost:3000`.

## 🐳 التشغيل السريع والموحد باستخدام Docker Compose

الحل الأسهل والجاهز للإنتاج بضغطة زر واحدة (بما في ذلك عتاد FFmpeg والـ Python):

```bash
# بداخل المجلد الجذري للمشروع قم بتشغيل الأمر التالي:
docker-compose up --build
```

سيقوم Compose بالعمليات التالية:
1. بناء وتدشين حاوية الـ FastAPI (وتنزيل نموذج WhisperX المناسب تلقائياً في أول عملية تفريغ).
2. بناء وتدشين حاوية موقع الويب Next.js وربطه بالسيرفر.
3. مشاركة المجلدات المؤقتة الخاصة بالرفع وحجم الملفات لتفادي امتلاء الذاكرة العشوائية.

## 🎯 تفعيل جودة الأداء ودعم بطاقة الرسوميات (GPU / CUDA Optimization)

بشكل افتراضي، تم تهيئة الكود ليعمل على المعالج (CPU) لتفادي الانقطاعات إذا لم تكن تمتلك بطاقة Nvidia. للتفعيل الفائق والحصول على سرعة ترجمة تصل إلى **10 أضعاف** باستخدام كروت شاشة Nvidia:

1. تأكد من تثبيت [NVIDIA Container Toolkit](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/latest/install-guide.html).
2. قم بتعديل قيمة المتغير `DEVICE` في ملف `docker-compose.yml` إلى `cuda`.
3. قم بإضافة قسم الـ `deploy` الخاص بإتاحة استخدام عتاد كروت الشاشة للحاويات:

```yaml
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: all
              capabilities: [gpu]
```

## 📋 هيكلية وإخراج الملفات المنتجة

بعد انتهاء المعالجة التلقائية والتوقيت الدقيق باستخدام WhisperX، يقدم الـ API الروابط التالية لتنزيل الملفات المنتجة لكل ملف وسائط:

- **رابط ملف النص الكامل (.txt):** يحتوي على النص الكامل مدمجاً بدون توقيتات للاستفادة الكاملة من القراءة السريعة والمستندات.
- **رابط ملف الترجمة القياسي (.srt):** مصمم ومعد لمطابقة أحدث مشغلات الميديا (مثل VLC).
- **رابط ملف ترجمة الويب (.vtt):** يتضمن ترويسة `WEBVTT` والرموز العشرية لتصميم دفق متزامن فوري على مشغلات الفيديو لمتصفحات كروم وسفاري وباقي المواقع.
